## 1. 产品定位

HxA Dash 是 HxA 系列套件之一，定位为 **Agent 团队信息面板**。

核心目标：让人机团队（Human × Agent）的分工、状态、工作流和任务进度实现**可视化**和**量化**，解决人类对 Agent 工作不透明的焦虑。

### 设计原则

> "员工/工作可视化增加人类安全感"

人类需要看到团队成员（包括 Agent）在做什么、做了什么、接下来做什么。透明度 = 信任。

### 与 HxA 系列关系

| 套件 | 定位 |
|------|------|
| HxA Connect | Agent 通讯与协作平台 |
| HxA Link | Agent 社交与服务平台 |
| **HxA Dash** | **Agent 团队状态面板** |

HxA Dash 从 HxA Connect 和 GitLab 获取数据，聚合展示。不做任务管理（那是 Connect/GitLab 的事），只做**信息展示和可视化**。

---

## 2. 目标用户

- **团队管理者**（如 Kevin）：需要一目了然地了解所有 Agent 的状态和产出
- **直播/展示场景**：向外部展示人机团队的工作方式和实时状态
- **团队成员**：了解其他成员（含 Agent）的工作进展

## 3. 功能需求

### 3.1 v1 — 最小可用版本（2026-03-13 直播可用）

#### 3.1.1 团队总览

每个 Agent/成员一张卡片，展示：

| 字段 | 来源 | 说明 |
|------|------|------|
| 名称 | 配置 | Agent 名称 |
| 角色 | 配置 | 如「全栈工程师」「前端工程师」「安全专家」 |
| 在线状态 | HxA Connect API | 在线 / 离线 / 最后在线时间 |
| 当前任务 | 配置/手动 | 当前正在做的事情描述 |
| 头像 | 配置 | 可选 |

**交互**：
- 卡片按在线状态排序（在线优先）
- 在线状态用颜色指示（绿色在线、灰色离线）
- 点击卡片可展开查看详细信息

#### 3.1.2 工作流看板

三列看板：

| 列 | 说明 |
|----|------|
| 待办 | 已分配但未开始的任务 |
| 进行中 | 正在执行的任务 |
| 已完成 | 近期完成的任务 |

每张任务卡片包含：
- 任务标题
- 负责人（Agent 名称）
- 来源（GitLab Issue/MR 链接，可选）
- 状态标签

#### 3.1.3 工作时间线

按时间倒序展示近期活动：
- GitLab MR 创建/合并
- GitLab Issue 创建/关闭
- 自定义事件（可通过 API 推送）

每条记录包含：时间戳 + 执行者 + 事件描述

#### 3.1.4 实时刷新

- 默认每 30 秒自动刷新数据
- 页面顶部显示「最后更新时间」
- 支持手动刷新按钮

## 4. 数据来源

### 4.1 HxA Connect API

```
GET /api/bots          → Agent 列表 + 在线状态
GET /api/bots/:name    → 单个 Agent 详情
```

认证：Bearer Token（agent_token）

### 4.2 GitLab API

```
GET /api/v4/groups/:id/merge_requests  → MR 活动
GET /api/v4/groups/:id/issues          → Issue 活动
```

认证：PRIVATE-TOKEN

### 4.3 配置文件

`config/team.json` — 团队成员配置：

```json
{
  "members": [
    {
      "name": "Jessie",
      "role": "技术负责人 / 协调",
      "avatar": "jessie.png",
      "connect_name": "Jessie",
      "gitlab_username": "Jessie",
      "current_task": "构建 HxA Dash v1"
    }
  ]
}
```

## 5. 技术方案

### 5.1 架构

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐
│   浏览器     │ ←── │  Node.js     │ ←── │ HxA Connect  │
│  (HTML/JS)  │     │  聚合服务    │ ←── │ GitLab API   │
└─────────────┘     │  :3479       │ ←── │ config/      │
                    └──────────────┘     └──────────────┘
```

### 5.2 技术选型

| 层 | 技术 | 理由 |
|----|------|------|
| 前端 | HTML + CSS + 原生 JS | 轻量、快速、无构建步骤 |
| 后端 | Node.js + Express | 聚合多数据源，提供统一 API |
| 部署 | PM2 + Caddy 反向代理 | 与现有基础设施一致 |

### 5.3 API 设计

后端提供以下 API：

| 端点 | 说明 |
|------|------|
| `GET /api/team` | 团队成员列表 + 在线状态 |
| `GET /api/tasks` | 任务看板数据 |
| `GET /api/timeline` | 工作时间线 |
| `POST /api/tasks` | 更新任务状态（可选） |

### 5.4 端口

- 服务端口：3479
- 路由：`jessie.coco.site/hxa-dash/` → `localhost:3479`
