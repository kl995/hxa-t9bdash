## 1. 产品定位

HxA Dash 是 HxA 系列套件之一，定位为 **Agent 团队信息面板**。

核心目标：让人机团队（Human × Agent）的分工、状态、工作流和任务进度实现**可视化**和**量化**。

### 双重价值（#1）

| 视角 | 价值 |
|------|------|
| **对人类** | 员工/工作可视化增加**安全感** — 随时看到 Agent 在做什么、做了什么、接下来做什么。透明度 = 信任。 |
| **对 Agent** | 增加**信息**，实现**有效协作** — Agent 通过面板了解其他成员的状态和工作，做出更好的协作决策。 |

HxA Dash 不只是给人看的监控面板，而是人机团队共享的信息中枢。

### 与 HxA 系列关系

| 套件 | 定位 |
|------|------|
| HxA Connect | Agent 通讯与协作平台 |
| HxA Link | Agent 社交与服务平台 |
| **HxA Dash** | **Agent 团队信息面板** |

HxA Dash 从 HxA Connect 和 GitLab 获取数据，聚合展示。定位是**信息可视化**，不重复任务管理功能。

---

## 2. 目标用户

- **团队管理者**（如 Kevin）：一目了然了解所有 Agent 状态和产出
- **Agent**：获取团队信息，支持协作决策
- **直播/展示场景**：向外部展示人机团队的工作方式和实时状态
- **团队成员（人类）**：了解其他成员（含 Agent）的工作进展

---

## 3. 功能架构（#2 分层可视化）

按层级组织可视化，从宏观到微观：

```
第一层：资源管理（Agent 管理）
  └── 团队全貌：有多少 Agent、各自角色、在线状态、资源分配
第二层：Agent 个体
  └── 单个 Agent 详情：当前任务、历史工作、能力、协作关系
第三层：项目与任务
  └── 工作流看板：项目进展、任务状态、负责人、时间线
```

## 4. 功能需求

### 4.1 v1 — 最小可用版本（2026-03-13 直播可用）

v1 目标：先做一个版本，找找感觉（#4）。覆盖三层可视化的基础功能。

#### 4.1.1 第一层：Agent 资源管理

团队总览面板，展示所有 Agent 的宏观状态：

| 字段 | 来源 | 说明 |
|------|------|------|
| 名称 | 配置 | Agent 名称 |
| 角色 | 配置 | 如「全栈工程师」「协调者」 |
| 在线状态 | HxA Connect API | 在线 / 离线 / 最后在线时间 |
| 当前任务 | 数据库 | 正在做什么（简要） |
| 头像 | 配置 | 可选 |

**交互**：
- 卡片按在线状态排序（在线优先）
- 在线状态用颜色指示（绿色在线、灰色离线）
- 点击卡片进入 Agent 个体视图

#### 4.1.2 第二层：Agent 个体视图

点击 Agent 卡片展开详情（#3 项目透视）：

| 模块 | 说明 |
|------|------|
| 基本信息 | 角色、能力标签、所属团队 |
| 当前工作 | 正在执行的任务列表 |
| 历史工作 | 近期完成的任务/MR/Issue |
| 协作关系 | 与哪些 Agent 在同一项目中协作（关系图） |

#### 4.1.3 第三层：项目与任务看板

三列看板：

| 列 | 说明 |
|----|------|
| 待办 | 已分配但未开始的任务 |
| 进行中 | 正在执行的任务 |
| 已完成 | 近期完成的任务 |

每张任务卡片包含：
- 任务标题
- 负责人（Agent 名称）
- 来源（GitLab Issue/MR 链接，可选）
- 状态标签

#### 4.1.4 工作时间线

按时间倒序展示近期活动：
- GitLab MR 创建/合并
- GitLab Issue 创建/关闭
- 自定义事件（可通过 API 推送）

每条记录包含：时间戳 + 执行者 + 事件描述

#### 4.1.5 实时刷新

- 默认每 30 秒自动刷新
- 页面顶部显示「最后更新时间」
- 手动刷新按钮

## 5. 数据来源

### 5.1 HxA Connect API

```
GET /api/bots          → Agent 列表 + 在线状态
GET /api/bots/:name    → 单个 Agent 详情（bio, role, tags, last_seen）
```

认证：Bearer Token（agent_token）

### 5.2 GitLab API

```
GET /api/v4/groups/:id/merge_requests  → MR 活动
GET /api/v4/groups/:id/issues          → Issue 活动
```

认证：PRIVATE-TOKEN

### 5.3 本地数据库（#4 持久化）

SQLite 存储：
- 团队成员配置（角色、头像、标签）
- 任务状态（看板数据）
- 工作时间线（事件记录）
- 协作关系快照

### 5.4 配置文件

`config/team.json` — 团队成员初始配置：

```json
{
  "members": [
    {
      "name": "Jessie",
      "role": "技术负责人 / 协调",
      "avatar": "jessie.png",
      "connect_name": "Jessie",
      "gitlab_username": "Jessie",
      "tags": ["Node.js", "协调", "产品"]
    }
  ]
}
```

## 6. 技术方案

### 6.1 架构

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐
│   浏览器     │ ←── │  Node.js     │ ←── │ HxA Connect  │
│  (HTML/JS)  │     │  聚合服务    │ ←── │ GitLab API   │
└─────────────┘     │  :3479       │ ←── │ SQLite DB    │
                    └──────────────┘     └──────────────┘
```

### 6.2 技术选型

| 层 | 技术 | 理由 |
|----|------|------|
| 前端 | HTML + CSS + 原生 JS | 轻量、快速、无构建步骤 |
| 后端 | Node.js + Express | 聚合多数据源，提供统一 API |
| 数据库 | SQLite（better-sqlite3） | 轻量持久化，无需额外服务 |
| 部署 | PM2 + Caddy 反向代理 | 与现有基础设施一致 |
| 认证 | Basic Auth | 与 jessie.coco.site 统一（#4） |

### 6.3 API 设计

| 端点 | 说明 |
|------|------|
| `GET /api/team` | 团队成员列表 + 在线状态 |
| `GET /api/team/:name` | 单个 Agent 详情（含历史工作、协作关系） |
| `GET /api/tasks` | 任务看板数据 |
| `POST /api/tasks` | 创建/更新任务 |
| `GET /api/timeline` | 工作时间线 |
| `POST /api/timeline` | 推送自定义事件 |

### 6.4 端口与路由

- 服务端口：3479
- 路由：`jessie.coco.site/hxa-dash/` → `localhost:3479`
- 认证：Basic Auth（coco/coco2026）
